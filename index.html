What It Does

The Scalable Thread Management Library is a C-based project that manages threads efficiently using a thread pool. It allows you to schedule tasks with priorities, handle task timeouts, and measure performance metrics like execution time and throughput. This library is ideal for applications that need to process multiple tasks concurrently while optimizing resource usage.

    Features

    Dynamic Thread Pool

    Create a configurable number of threads to handle tasks efficiently, reducing overhead and improving scalability.

    Priority-Based Scheduling

    Tasks are executed based on priority using a linked-list queue, ensuring high-priority tasks are processed first.

    Task Timeout

    Automatically cancels tasks that exceed a 2-second execution time, preventing resource bottlenecks.

    Performance Metrics

    Measures execution time and throughput of tasks, providing insights into the library's efficiency.

    How It Works

    The library uses a thread pool to manage a fixed number of worker threads. Tasks are added to a priority-based queue, and threads pick up tasks based on their priority. A timeout checker ensures tasks don’t run too long, and performance metrics track the system's efficiency.

    Process Flow

    Add Task → Priority Queue → Thread Pool → Execute → Timeout Check → Performance Metrics

    Usage Instructions

    Clone the repository:

    git clone https://github.com/rakhi120805/Scalable-Thread-Management-Library-.git

    Compile the project:

    gcc -o thread_manager thread_manager.c Thread.c MessageQueue.c Performance.c -pthread

    Run the project:

    chmod +x thread_manager
./thread_manager
